<p class="section-eyebrow">§ 1 · Premise</p>
## One attention is not enough

A single scaled dot-product attention head over the full $d_{\text{model}}$ space would be
asked to learn every kind of relation a token might need — syntactic, semantic, positional,
discourse-level — into one set of weights. The 2017 transformer's bet: it works better to run
$H$ independent attention heads in parallel, each free to attend to a different aspect of the
input.

This is the move that makes "attention is all you need" actually work. A single attention
flattens the relational structure of language into one inner-product space; many parallel
heads let that structure decompose.

<p class="section-eyebrow">§ 2 · The math</p>
## Project, attend, concatenate

Per head $i \in \{1, \ldots, H\}$, project Q, K, V into a $d_h$-dimensional subspace (where
$d_h = d_{\text{model}} / H$):

$$\mathbf{Q}_i = X W_Q^{(i)},\quad \mathbf{K}_i = X W_K^{(i)},\quad \mathbf{V}_i = X W_V^{(i)},$$

run scaled dot-product attention:

$$\mathrm{head}_i = \mathrm{softmax}\!\left(\frac{\mathbf{Q}_i \mathbf{K}_i^\top}{\sqrt{d_h}}\right)\mathbf{V}_i,$$

concatenate and project back:

$$\mathrm{MHA}(X) = [\mathrm{head}_1; \ldots; \mathrm{head}_H]\, W_O.$$

```python
def mha(x, W_Q, W_K, W_V, W_O, H, mask=None):
    # x: [B, T, d_model]
    B, T, D = x.shape
    d_h = D // H
    q = (x @ W_Q).view(B, T, H, d_h).transpose(1, 2)  # [B, H, T, d_h]
    k = (x @ W_K).view(B, T, H, d_h).transpose(1, 2)
    v = (x @ W_V).view(B, T, H, d_h).transpose(1, 2)
    logits = (q @ k.transpose(-2, -1)) / d_h**0.5
    if mask is not None: logits = logits.masked_fill(mask, float("-inf"))
    out = logits.softmax(-1) @ v                       # [B, H, T, d_h]
    return out.transpose(1, 2).reshape(B, T, D) @ W_O.T
```

<p class="section-eyebrow">§ 3 · Why it was eventually replaced</p>
## The KV-cache problem

MHA has one cost that becomes intolerable at long context: each head has its own K, V. The KV
cache per token per layer is $2 \cdot H \cdot d_h = 2 \cdot d_{\text{model}}$ floats. For a 64-head
model at $d_{\text{model}} = 8192$, that is 32 KB per token per layer in fp16. At 128K context
across 80 layers, the cache exceeds 300 GB — well past the practical inference budget.

The successor lineage tightened this constraint:

- **MQA** (Shazeer 2019): one K, V shared across all heads. $H\times$ cache reduction, mild
  quality cost.
- **GQA** (Ainslie et al. 2023): partial sharing across groups. Most of MQA's savings without
  the quality cost. See the [GQA entry](/llm-arch-kb/attention/gqa/).
- **MLA** (DeepSeek 2024): compress K, V to a per-token latent. 5-7× smaller cache than MHA
  and on-par quality. See the [MLA entry](/llm-arch-kb/attention/mla/).

For dense decoders shipping in 2024–26, plain MHA is essentially never used — every entry on
this site's adoption list runs GQA or MLA instead. That is why this entry's `adopted_by` list
is empty: MHA's role is historical.

<p class="section-eyebrow">§ 4 · What MHA still tells us</p>
## The conceptual frame

Two things from the 2017 paper remain load-bearing:

1. **The scaled-dot-product attention kernel.** Every variant since — MQA, GQA, MLA, linear,
   lightning — still computes some softmax over $\mathbf{Q}\mathbf{K}^\top$ before mixing values.
   The shape of the inner-product attention has not been displaced.
2. **The per-head subspace decomposition.** Multi-head specialization survives in every
   descendant; the head count $H$ remains the unit of parallelism. The variants only change
   how K and V are *shared* across heads, not whether heads exist.

<Summary>
**MHA in one breath.** Run H independent attention heads in d_h-dim subspaces, concatenate, project.
The KV cache is the cost that killed it for long-context production — every modern decoder
shares K and V more aggressively (GQA, MLA) — but the heads-as-specialization framework
remains the substrate every variant builds on.
</Summary>
