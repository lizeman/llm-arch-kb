<p class="section-eyebrow">§ 1 · The choice</p>
## Where does the norm go?

For a transformer block with sub-layer $f$ (attention or FFN), there are three places the
LayerNorm/RMSNorm can sit relative to the residual addition:

$$
\begin{aligned}
\text{Post-Norm:}\quad & \mathbf{x}_{\ell+1} = \mathrm{Norm}\bigl(\mathbf{x}_\ell + f(\mathbf{x}_\ell)\bigr)\\
\text{Pre-Norm:}\quad  & \mathbf{x}_{\ell+1} = \mathbf{x}_\ell + f\bigl(\mathrm{Norm}(\mathbf{x}_\ell)\bigr)\\
\text{Sandwich:}\quad  & \mathbf{x}_{\ell+1} = \mathbf{x}_\ell + \mathrm{Norm}_2\bigl(f(\mathrm{Norm}_1(\mathbf{x}_\ell))\bigr)
\end{aligned}
$$

The 2017 original transformer used Post-Norm. Almost every open LLM since 2020 uses Pre-Norm.

<p class="section-eyebrow">§ 2 · Why Pre-Norm won</p>
## A clean residual stream

In Pre-Norm the residual addition is on a stream that never gets renormalized in place. The
forward signal $\mathbf{x}_\ell$ moves through the network linearly; each block only adds a
normalized perturbation. Mathematically this means the gradient of the loss with respect to
$\mathbf{x}_\ell$ has the form $\frac{\partial L}{\partial \mathbf{x}_{\ell+1}}(I + \text{stuff})$
— the identity dominates, gradients do not vanish or explode with depth, and warmup becomes
unnecessary.

Xiong et al. (2020) prove this gradient stability formally and show that Post-Norm transformers
without warmup diverge at depth 12+ on translation tasks, while Pre-Norm trains stably at
depth 24+ without any warmup at all (Table 1).

<p class="section-eyebrow">§ 3 · What Post-Norm gave up</p>
## A tradeoff in expressivity

Post-Norm's appeal was that each layer's output is renormalized into a constrained distribution
before being passed on. Some analyses (DeepNet, 2022; OLMo 2, 2025) argue that Pre-Norm allows
the residual-stream norm to grow unboundedly across depth, making downstream layers harder to
condition. The fix is either:

- Add a final norm at the network output (universal in modern designs), or
- Switch to a hybrid: sandwich placement, or OLMo 2's reversion to Post-Norm with careful init.

```python
# Pre-Norm — the default
def block_pre(x, attn, ffn, norm1, norm2):
    x = x + attn(norm1(x))
    x = x + ffn(norm2(x))
    return x

# Sandwich — Gemma-3 style "norm everywhere"
def block_sandwich(x, attn, ffn, n1, n2, n3, n4):
    x = x + n2(attn(n1(x)))
    x = x + n4(ffn(n3(x)))
    return x
```

<p class="section-eyebrow">§ 4 · Modern variations</p>
## Sandwich and norm-everywhere

Sandwich-LN (CogView, Ding et al. 2021) and Gemma 3's "norm everywhere" both add a *second*
norm inside each residual branch — applied after the sub-layer output but still inside the
residual. This caps the magnitude of the perturbation each block adds, keeping the residual
stream from blowing up at depth without giving up Pre-Norm's gradient-flow properties.

Gemma 3 uses this aggressively across its 27B and smaller variants. OLMo 2 takes the
alternative route — reverting to Post-Norm with a carefully chosen initialization — and reports
training stability at long horizons.

For the dominant 2024–26 open stack (Llama, DeepSeek, Qwen, Mistral families), plain Pre-Norm
remains the consensus choice. The norm-everywhere and Post-Norm-revival branches are minority
options, both empirically defensible.

<Summary>
**Norm placement in one breath.** Pre-Norm cleared the path for stable deep training and is the
default. Sandwich placements (Gemma 3) add a second norm inside each branch to cap perturbation
size. Post-Norm is mostly historical, with OLMo 2 as a deliberate revival.
</Summary>
